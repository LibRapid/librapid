\end{DoxyItemize}\hypertarget{md_librapid__r_e_a_d_m_e_autotoc_md5}{}\doxysubsection{Multithreading}\label{md_librapid__r_e_a_d_m_e_autotoc_md5}
By default, Lib\+Rapid will automatically run sufficiently large loops in paralle, however, the branch required for this dynamic selection can cause a very slight performance hit with smaller arrays. For this reason, if you know you\textquotesingle{}ll only be dealing with relatively small arrays (500x500 or smaller), it might make sense to disable this.

To do so, simply define {\ttfamily LIBRAPID\+\_\+\+OPTIMISE\+\_\+\+SMALL\+\_\+\+ARRAYS} before your {\ttfamily \#include \texorpdfstring{$<$}{<}librapid\texorpdfstring{$>$}{>}}

For more multithreading options, check out the {\ttfamily librapid\+::global} namespace in the documentation.\hypertarget{md_librapid__r_e_a_d_m_e_autotoc_md6}{}\doxysubsection{Debug Information}\label{md_librapid__r_e_a_d_m_e_autotoc_md6}
When compiling Lib\+Rapid in Release mode, you may want to enable the error checking and logging. It\textquotesingle{}s enabled by default in debug mode.

To enable this, define {\ttfamily LIBRAPID\+\_\+\+ENABLE\+\_\+\+ASSERT} before including Lib\+Rapid.
